## Changes

### New: Default AI Audit System

A new document-based AI audit system that processes from `transaction_audit_history` queue using OpenRouter (Grok `x-ai/grok-4.1-fast`) instead of Gemini.

**Key features:**
- Evidence document classification (5 types: waste photos, daily/monthly weight forms, weight tickets, sales receipts)
- Structured data extraction from classified documents
- Fuzzy matching of extracted data against transaction and record fields
- 2-level threading: Level 1 per org, Level 2 per transaction (max 10 workers, max 25 transactions/batch)
- Per-record audit results (`ai_audit_status`, `ai_audit_note` on `transaction_records`)
- Only processes transactions with `transaction_method IN ('origin', 'qr_input')`

### New: 3-Tier Recycling Rate Calculation

Refactored recycling rate calculation in Overview and Performance reports to integrate traceability data instead of relying solely on material category classification.

**3-tier approach:**
1. **Tier 1 (Fully Traced)**: Record has `traceability_group_id` AND group has 100% completion → recyclable weight determined by leaf transport transaction disposal methods weighted by `absolute_percentage`
2. **Tier 2 (Partially Traced)**: Record has `traceability_group_id` BUT group is NOT fully complete → traced portion uses disposal methods, untraced portion falls back to category
3. **Tier 3 (No Traceability)**: Record has NO `traceability_group_id` → old category-based method (`category_id in {1, 3}` = recyclable)

**Disposal method classification:**
- **Diverted from Disposal** (recyclable): `Preparation for reuse`, `Recycling (Own)`, `Other recover operation`, `Recycle`
- **Directed to Disposal** (non-recyclable): `Composted by municipality`, `Municipality receive`, `Incineration without energy`, `Incineration with energy`

**API response additions:**
- `traceability_fully_managed` (boolean) added to Overview and Performance report responses
- `true` only when ALL records have a traceability group with 100% completion
