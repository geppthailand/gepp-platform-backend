# GEPP Platform - Database Model Changes v1.1.0

**Date:** March 14, 2026
**Database:** PostgreSQL
**ORM:** SQLAlchemy
**Base:** v1.0.0_20260308

---

## Summary

v1.1.0 adds support for 3-tier recycling rate calculation by introducing a reverse pointer from `transaction_records` to `traceability_transaction_group`, and adds per-record AI audit fields. It also introduces document-based AI audit configuration tables.

---

## New Tables

### ai_audit_document_types

Evidence document type definitions for the default AI audit system.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| name | VARCHAR(100) | NO | — | Document type name (e.g., "waste_photos", "weight_ticket") |
| description | TEXT | YES | NULL | Human-readable description |
| extractable_fields | JSONB | YES | `[]` | Fields that can be extracted from this document type |

**Seeded records:** 5 document types (waste photos, daily weight forms, monthly weight forms, weight tickets, sales receipts)

### organization_audit_doc_require_types

Per-organization required document types for AI audit.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| organization_id | BIGINT FK → organizations(id) | NO | — | Organization |
| audit_document_type_id | BIGINT FK → ai_audit_document_types(id) | NO | — | Required document type |
| level | VARCHAR(50) | NO | — | `'transaction'` or `'record'` |
| is_required | BOOLEAN | NO | TRUE | Whether this document type is required |

### organization_audit_check_columns

Per-organization field verification configuration for AI audit matching.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| organization_id | BIGINT FK → organizations(id) | NO | — | Organization |
| level | VARCHAR(50) | NO | — | `'transaction'` or `'record'` |
| column_name | VARCHAR(100) | NO | — | Field name to check |
| check_type | VARCHAR(50) | NO | — | Match type (e.g., `'exact'`, `'fuzzy'`, `'range'`) |
| threshold | DECIMAL(5,2) | YES | NULL | Match threshold (0-100) |

## Entity Relationship Changes

```
TransactionRecord ──N:1──→ TraceabilityTransactionGroup  (NEW: traceability_group_id)
    Previously: group referenced records via ARRAY column transaction_record_id
    Now: bidirectional — group has ARRAY of record IDs, records have FK back to group
```

## Key Design Notes

### traceability_group_id (Reverse Pointer)

The `traceability_transaction_group` table already stores an array of `transaction_record_id`s. The new `traceability_group_id` column on `transaction_records` provides the reverse direction for efficient joins in report queries without needing `ANY(array_column)` lookups.

- **Set automatically** during `_upsert_traceability_groups_for_transaction()` in `transaction_service.py`
- **Backfill existing data** via `POST /api/debug/traceability/backfill_group_ids`
- **Used by** `recycling_rate_helper.py` to determine which tier of recycling rate calculation applies to each record
