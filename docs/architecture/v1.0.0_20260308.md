**Date**: March 8, 2026
**Version**: 1.0.0

## 1. System Overview

GEPP (Green Environment & Plastic Platform) is a comprehensive waste management and ESG (Environmental, Social, and Governance) reporting platform designed for organizations in Thailand to track, manage, and report waste transactions with AI-powered auditing capabilities.

**Mission**: "One Click Waste Management Data in Thailand"

**Core Capabilities**:
- Waste transaction tracking and management
- AI-powered waste image auditing (Vertex AI / Gemini Pro Vision)
- GRI 306 sustainability compliance reporting
- Extended Producer Responsibility (EPR) management
- Material traceability and chain of custody
- Multi-tenant organization management with RBAC
- Reward/gamification system
- QR code mobile input for field workers
- Knowledge management with vector search
- Real-time notifications and alerts

## 4. Authentication Architecture

### Token Lifecycle
```
┌─────────┐     POST /api/auth/login     ┌──────────┐
│  Client  │ ───────────────────────────→ │  Backend │
│          │ ←─────────────────────────── │          │
│          │  { auth_token, refresh_token }│          │
└────┬─────┘                              └──────────┘
     │
     │  Authorization: Bearer {auth_token}
     │  (every API request)
     │
     │  On 401 → POST /api/auth/refresh
     │  { refresh_token } → new auth_token
```

### Token Types
| Token | Lifetime | Use Case |
|-------|----------|----------|
| Auth Token | 1 day | Standard API access |
| Refresh Token | 7 days | Token renewal |
| Device Token | 1 day | IoT device access |
| Integration Token | Custom | Service-to-service |

### Password Security
- bcrypt hashing with salt rounds
- Password reset via email with JWT token
- Email verification flow

## 5. AI Audit Pipeline

```
┌───────────┐     ┌──────────────┐     ┌────────────────┐
│ Transaction│     │  AI Audit    │     │   Vertex AI    │
│  Created   │────→│  Service     │────→│  Gemini Pro    │
│ (+ images) │     │              │     │  Vision        │
└───────────┘     └──────┬───────┘     └───────┬────────┘
                         │                      │
                         │  Phase 1: Extraction  │
                         │  - Analyze image       │
                         │  - Extract materials   │
                         │  - Identify weights    │
                         │  - Detect anomalies    │
                         │                      │
                         │  Phase 2: Judgment     │
                         │  ←──────────────────  │
                         │  - Apply audit rules   │
                         │  - Cross-reference DB  │
                         │  - Generate verdict    │
                         ▼
                  ┌──────────────┐
                  │ Pattern      │
                  │ Matching     │
                  │ Service      │
                  └──────┬───────┘
                         │  Match org's response patterns
                         │  Render Thai message template
                         ▼
                  ┌──────────────┐
                  │ Transaction  │
                  │ Updated      │
                  │ ai_audit_note│
                  │ ai_audit_    │
                  │ status       │
                  └──────────────┘
```

### Prompt System
- LangChain YAML templates in `prompts/ai_audit/`
- 2 rule sets: `default`, `bma`
- 4 waste types each: general, recyclable, organic, hazardous
- PromptLoader utility for dynamic template loading

### Audit Results
- Stored in `transactions.ai_audit_note` (JSONB)
- Token usage tracked in `transactions.audit_tokens` (JSONB)
- Status: queued → approved/rejected/no_action/failed

## 6. Multi-Tenant Architecture

```
Organization (tenant boundary)
├── OrganizationInfo (company details)
├── OrganizationSetup (hierarchy structure)
│   ├── root_nodes (JSON tree)
│   └── hub_node (JSON)
├── OrganizationRoles (RBAC)
│   └── OrganizationPermissions (M:M)
├── Subscription (plan limits)
├── UserLocations (users + locations)
│   ├── is_user=true (people)
│   └── is_location=true (sites)
├── Materials (org-specific + global)
├── Transactions
├── InputChannels (QR codes)
├── LocationTags & Tenants
└── AI Audit Configuration
    ├── ai_audit_rule_set_id
    ├── enable_ai_audit_response_setting
    └── AiAuditResponsePatterns
```

### Data Isolation
- All queries filter by `organization_id`
- Materials can be `is_global=true` (shared) or org-specific
- Audit rules can be `is_global=true` or org-specific
- Soft delete via `deleted_date` column (NULL = active)

## 7. Data Flow: Transaction Lifecycle

```
1. CREATE    → Transaction created (status: draft/pending)
              → TransactionRecords linked
              → Images uploaded to S3

2. AI AUDIT  → Images sent to Vertex AI
              → Audit rules evaluated
              → Response patterns matched
              → Status: approved/rejected

3. MANUAL    → Human auditor reviews
   AUDIT       → Audit notes recorded
              → Status updated

4. COMPLETE  → Transaction finalized
              → GHG calculations applied
              → Reports updated
              → Traceability chain extended

5. REPORT    → Aggregated in Reports service
              → GRI 306 compliance calculated
              → PDF export available
```

## 10. Security Measures

- JWT token-based authentication with refresh mechanism
- bcrypt password hashing
- CORS configuration (disabled by default)
- Soft-delete pattern (data never permanently removed)
- Organization-level data isolation
- Role-based access control (RBAC)
- S3 presigned URLs for secure file access (time-limited)
- Integration tokens with custom secrets

## 12. Scalability Considerations

- **Serverless**: Lambda auto-scales with request volume
- **Database**: RDS with read replicas if needed
- **CDN**: CloudFront for global frontend delivery
- **File Storage**: S3 with virtually unlimited storage
- **AI**: Vertex AI scales independently
- **Connection Pooling**: SQLAlchemy QueuePool prevents connection exhaustion
