OpenRouter Migration Plan

Replace direct Gemini API calls with OpenRouter for multi-model access, cost optimization, and fallback resilience.

Why OpenRouter?

  • Multi-model access — Use Claude, GPT-4o, Gemini, Llama from a single API
  • Automatic fallback — If one provider is down, route to another
  • Cost optimization — Choose cheaper models for simple tasks, powerful models for complex
  • Unified billing — Single API key, one invoice
  • Model comparison — A/B test different models on same prompts

Migration Steps

# Current: Direct Gemini via LangChain from langchain_google_genai import ChatGoogleGenerativeAI model = ChatGoogleGenerativeAI(model="gemini-2.0-flash", api_key=GEMINI_KEY) # Target: OpenRouter via LangChain OpenAI-compatible endpoint from langchain_openai import ChatOpenAI model = ChatOpenAI( model="google/gemini-2.0-flash-001", # or "anthropic/claude-3.5-sonnet" api_key=OPENROUTER_KEY, base_url="https://openrouter.ai/api/v1", default_headers={"HTTP-Referer": "https://geppdata.com"} )

Anomaly Detection Architecture

Multi-layer anomaly detection combining statistical analysis, temporal patterns, and AI reasoning.

Detection Layers

Layer 1: Statistical (Real-time)

  • Z-score outlier detection on weight/quantity
  • IQR (Interquartile Range) for price validation
  • Per-material baseline: mean ± 2σ
  • Per-location volume baseline
  • Rolling 30-day moving average
  • Runs on every transaction insert/update

Layer 2: Temporal (Batch)

  • Transaction frequency anomalies
  • Time-of-day patterns (unusual hours)
  • Day-of-week deviation
  • Seasonal trend breaks
  • Sudden volume spikes/drops
  • Runs every 6h as cron job

Layer 3: Cross-Entity (Daily)

  • Duplicate transaction detection
  • Same-day same-material same-weight
  • Cross-location volume consistency
  • Material ratio drift detection
  • Evidence reuse detection (image hash)
  • Runs once per day

Layer 4: AI Reasoning (On-demand)

  • LLM reviews flagged transactions
  • Context-aware severity assessment
  • Natural language anomaly explanation
  • Recommendation for reviewer
  • Triggered when statistical flags accumulate
